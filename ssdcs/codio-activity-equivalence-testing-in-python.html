Exploring Linters to Support Testing in Python: Question 1

When I compiled and ran the code, the programme crashed with the following error message:
File "styleLint.py", line 5
""" Return factorial of n """
IndentationError: expected an indented block

Based on this error message, I have concluded that the fix for the bug causing it is actually quite simple. The problem in this case is the incorrect, or rather missing indentation (required in Python).

In order to fix this bug, I added indentation to the comment on line 5, as well as proper indentation within the function itself.

Exploring Linters to Support Testing in Python: Question 2

I tried to install pylint, but it kept throwing an error. I managed to find a solution on Stackoverflow, which was to upgrade pip:
python -m pip install --upgrade pip
Afterwards, I successfully installed pylint.

pylint served as a great ally in this process, as it pinpointed to every problem in the code and even offered solutions to some of them. I am proud to say that I fixed every single issue detected by pylint, as you may see in the second screenshot.

Exploring Linters to Support Testing in Python: Question 3

Error report produced by flake8 is in many ways similar to the one from pylint. It differs from pylint in its structure. I found the flake8 report to be more readable, it offers error codes, which can be looked up on their website. For example the most common error code that it produced in this report was E111, which means that the indentation is not a multiple of 4.
Explanations and examples can be found on the following link: https://www.flake8rules.com/rules/E111.html

I received a rather large amount of errors after running flake8 on the metricTest.py file. This file had line numbers on every line.
I managed to solve almost all the problems - only one of them persisted (visible in the screenshot).
You can find the code, i.e. my amendments to it on the following link:
https://github.com/savanovic-essex/testing-with-python/blob/master/metricTest.py
