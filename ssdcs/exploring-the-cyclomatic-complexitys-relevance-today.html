Exploring the Cyclomatic Complexity’s Relevance Today

Thomas McCabe developed the cyclomatic complexity metric to assess a program's stability and degree of trust. Cyclomatic complexity counts the number of linearly distinct pathways that can be taken through a program module. It reveals how intricate the code is. One may quickly determine the complexity of the code based on the number provided for each method in the source code. Cyclomatic complexity is strongly correlated with the number of lines of code (Meneely, 2016). Therefore, it should not be used in testing the validity of code design today because it doesn't add any new information about bug prediction (Meneely, 2016). Cyclomatic complexity offers relatively little novel information that is actionable but often not useful, in contrast to the more straightforward to gather the number of lines of codes.

Cyclomatic complexity is no longer relevant today because it is based on inadequate theoretical underpinnings and an inadequate model of software development (Meneely, 2016). Additionally, it appears to be little more than a proxy for large software classes and is frequently outperformed by lines of code. Developing secure software is not relevant in the quest since more complex software systems increase the likelihood of security problems. Complexity makes it challenging for engineers to comprehend the software (Alenezi & Zarour, 2020), and this impacts the system's maintainability and makes it challenging to introduce upgrades.

Additionally, software complexity leads to vulnerable software, making it challenging to develop secure software. Sometimes an increase in complexity is the primary cause of vulnerabilities in a software system. Complexity and exposure have a hazy relationship that makes generalizations impossible. Various security experts have studied and contributed to the security measures necessary for developing secure software (Alenezi & Zarour, 2020). However, developers either remain ignorant of it or introduce deliberate complexities in the system to avoid rewriting the codes.
